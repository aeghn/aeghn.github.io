1.2. ideapad-conservation-mode

对于锂电池来说，完全耗尽电池或者尝试充满电池都可能会影响电池健康。为了维护电池健康，将电池维持在 60% 左右是一个比较好的实践。联想的部分笔记本提供了这一功能，被称为 ideapad-conservation-mode 。

通过以下几步，就可以开始该功能。

  1. 确认开启 ideapad_laptop 内核模块。 

    $ lsmod | grep ideapad_laptop

# modprobe ideapad_laptop    # 没有开启的话，尝试手动开启。

  2. 控制该功能的文件是 /sys/bus/platform/drivers/ideapad_acpi/VPC2004:00/conservation_mode

    如果值为 1 就代表开启了电池保护模式，如果值为 0 代表关闭了电池保护模式。

  3. 也可以通过 tlp 来实现该功能，tlp 实际上也是通过该文件实现的此功能。

2. Windows 相关

2.1. QEMU/KVM

2.1.1. 基础

  1. 直通磁盘到虚拟器
    1. 优缺点比较

      非必要，但是如果直通磁盘的话有这些好处：

      • 更好的性能 虽然如果使用 qcow2 可以获得不错的性能，但是直接使用物理磁盘的话，可以获得更加强悍的性能。
      • 只需要维护一套上下文环境 如果 GNU/Linux 发生故障，或者需要物理使用 Windows 时，那么是和虚拟机中的系统一致的，不需要维护第二套 Windows 环境，减少脑力切换，系统维护和存储。

      但是在我的使用中也发现了一些问题，比如，物理机启动 Windows 时，有时候会提示正在准备设备，可能需要考虑如何解决。

    2. 在物理硬盘上安装 Windows 系统

      因为我使用了直通磁盘的方式，所以我直接使用物理机安装系统即可。

      我选择了 =Windows 10 Ltsc 2019=，因为该系统支持时间够长，达到惊人的十年。

      需要注意的是，如果像我一样双硬盘的话， Windows 可能使用另一块硬盘上的 EFI 分区（如果存在的话），我的做法是，提前分出来 500M，在安装结束后，将该分区设置为 EFI 分区，并且将 Windows 的引导文件迁移到该分区内。

    3. 直通方式
      1. 获取硬盘 id

        直通硬盘时使用硬盘 id 可以保证唯一性，避免因为重启等问题导致的硬盘入口（说的是 /dev/sdX 的形式）发生变化。 

        $ ls -l /dev/disk/by-id/*
================
/dev/disk/by-id/ata-ST1000LM002-9VQ14L_Z0501SZ9 -> ../../sdd
      2. 在 virt-manager 中添加

        在创建虚拟机或者后面添加硬盘过程中直接填入上一步获取的值。

  2. 直通 Nvidia 显卡到虚拟机
    1. 使用 vfio-pci 接管 Nvidia 设备
      1. 获取 Nvidia 设备的厂家和设备编码

        使用 lspci1 来获取 Nvidia 显卡厂家和产品编码： 

        lspci -nnk | grep -i -A 2 NVIDIA
      2. 使用 vfio-pci 驱动管理 Nvidia 设备

        首先需要确保 modprobe2 加载 vfio-pci 模块，然后需要告知 vfio-pci 需要管理哪些设备。在 /etc/modprobe.d/ 创建如下配置文件并写入： 

        /etc/modprobe.d/vfio-nvidia.conf
=========================
options vfio-pci ids=10de:
      3. 确保 vfio 抢先 nvidia 或者 nouveau 等驱动获取独显
        1. 使用 mkinitcpio 加载 vfio-pci 驱动

          在 /etc/mkinitcpio.conf 中的 MODULES 中添加如下内容： 

          MODULES=(vfio_pci vfio vfio_iommu_type1 vfio_virqfd)

          mkinitcpio3 用来创建 initial ramdisk4 环境，在 init 程序获取控制权之前？。并且 MODULES 中存放的是需要在任何启动钩子运行前需要加载的内核模块（ Kernel modules to be loaded before any boot hooks are run. ），因此我们可以确保 vfio-pci 驱动可以在 nvidia 驱动之前获取 Nvidia 设备的控制权。

        2. 重新生成 initramfs

          修改 /etc/mkinitcpio.conf 之后需要重新生成 =initramfs=，使新配置生效。 

          # mkinitcpio -P
      4. 至此，主机层面的设置就结束了

  1. 使用 rdp 访问虚拟机
    1. 概述

      使用 rdp 更加方便，很多需要自己折腾的，Windows 都提供了解决方案。但是图形性能不够好，即使根据微软官方的文档5修改注册表后，我这也只能维持在 30FPS 左右（通过 TestUFO.com 进行测试），我的客户端使用的是 FreeRDP，该软件只实现了 TCP 协议，可能实现了 UDP 协议以后可以获得更好的性能。

      1. 破解 30 帧限制

"DWMFRAMEINTERVAL"=dword:0000000f
      2. 使用 H264/AVC444
    3. 连接命令
      1. 启动命令
        #!/usr/bin/env bash

LOGIN_INFO=("/v:192.168.122.38" "/cert-ignore" "/u:***" /p:"***")
VIDEO_INFO=( /gfx:avc444 /gfx-progressive /gdi:hw)
GRAPHIC_INFO=(/dynamic-resolution /glyph-cache)
RFX_INFO=(/rfx /rfx-mode:video)

SOUND_INFO=(/sound:sys:pulse /microphone)
OTHER_INFO=(+clipboard +fonts +menu-anims +window-drag +aero /multitransport /network:auto)

freerdp() {
    wlfreerdp "${LOGIN_INFO[@]}" "${VIDEO_INFO[@]}" "${GRAPHIC_INFO[@]}" "${RFX_INFO[@]}" "${SOUND_INFO[@]}" "${OTHER_INFO[@]}"
}

freerdp &!
